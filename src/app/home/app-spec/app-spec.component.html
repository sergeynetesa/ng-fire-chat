Test Application Specification Overview
This is a test app to prove technical skill in Web UI and Firebase / Firestore.
We want to build a simple one-to-many  Chat interface. In this case we have many End Users and 1 support Person who chats with them, a Therapist. The Users and Chat data are all stored in Firestore. All chat data should be loaded from Firestore.
All updates to Users and Chat text should be tracked with a document or collection snapshot listener (see https://firebase.google.com/docs/firestore/query-data/listen)
Screen
There is 1 screen with 3 main sections.

Section 1
This is the Current User Chat. It shows:
name of the Current User (e.g. User 2). This should update when a new user is selected.
The chat history.
These should be loaded from Firestore and kept up to date using snapshot listeners.
The text field for a new message.
The "User Send" button. When the send button is pressed, the text in the text field is saved to Firestore and the text field cleared.
Text is saved to the document /users/<UUID>/messages/<YYYY-MM-DD> a new field is saved for each text message. The field is a map with two entries "source":"user"" and "text":<message text> in addition the current timestamp is save to the document /users/<UUID> to the field "user_msg"
Section 2
This is the Therapist chat. It shows:
A fixed name (e.g. Therapist).
The chat history. These should be loaded from Firestore and kept up to date using snapshot listeners.
The text field for a new message.
"Therapist Send" button. When the send button is pressed, the text in the text field is saved to Firestore and the text field cleared.
Text is saved to the document /users/<UUID>/messages/<YYYY-MM-DD> a new field is saved for each text message. The field is a map with two entries "source":"therapist" and "text:<message text> in addition the current timestamp is save to the document /users/<UUID> to the field "therapist_msg"
Section 3
This shows: The User Selection List. This allows selection of the current user. It also displays if there are unread messages form the user or therapist.
A message is unread if the "user_msg" timestamp != the "therapist_msg" timestamp.
A "Create User" button when pressed it adds a new user to Firestore and the User List should update using a snapshot listener. New users are added to /users/<UUID> with the fields "name":String , "therapist_msg":Timestamp and "user_msg":Timestamp set. A ôMark Readö button. When pressed this button with set the current users "therapist_msg" field equal to the "user_msg" field. The newer of the two timestamps should used.
Firebase
You should start form an empty Firestore and build up the following structure:
  • Users (collection)
  • <UUID> (document)
  • name : String (field)
  • therapists_msg : Timestamp (field)
  • user_msg : Timestamp (field)
  • messages (collection)
  • <YYYY-MM-DD> (document)
  • HHMMSS.sss : Map (field)
  • source : String (map field)
  • text : String (map field)

NOTE: you only need to display the current day's chat text.
